Project 4: Building an AI Sports Coach with LLM and Formal Methods
1. Introduction
Traditionally, Probabilistic Model Checking (PMC) was used to analyze the correctness and
performance of computer systems and protocols. However, due to the expressiveness and
generality of PMC’s capabilities, it can be applied in other areas as well. For example, during
the lecture, we demonstrated an interesting case where PMC is applied to tennis analytics to
reason about the relationship between a player's strategy and his winning probabilities. This
example models a tennis singles tiebreaker game. Using this model, we are able to predict
who will win the match and suggest player strategies to improve their chances of winning.
The lecture demonstrated how to manually build and check a model; this project requires
you to build an "AI Sports Agent" that automates this process. You will integrate Large
Language Models (LLMs) with Formal Verification (PAT) to create a system that can answer
complex strategic questions.
Traditional LLMs (like GPT or Gemini) are excellent at natural language but unreliable at
precise probability calculation ("hallucination" and “black-box”). Formal methods are perfect
for calculation but lack flexible interfaces. Your goal is to bridge this gap: the LLM will act as
the "Coach Interface," translating natural language queries into executable PCSP# models,
running the verification, and synthesizing the mathematical results into actionable coaching
advice.
2. The System Architecture
Your system will operate as a Tool-Using Agent. The LLM does not "guess" the answer; it
builds a formal model to calculate the answer.
The workflow you must implement is:
1. User Query: (e.g., "How can Player A improve their serve against Player B?")
2. LLM Reasoning: The LLM analyzes the query, identifies the necessary data, and
determines the type of analysis (Reachability vs. Sensitivity).
3. Tool Execution (The "Hands"):
○ Data Retrieval: Scripts fetch historical stats for the specific players
mentioned.
○ Model Construction: The system dynamically injects these probabilities into
a parametric PCSP# template.
○ Verification: The system runs the PAT model checker (via command
line/API) to get the precise probability.
4. Result Synthesis: The LLM reads the PAT output (e.g., "Win rate increases from
0.62 to 0.69") and formulates the advice.
3. Project Tasks
Part A: The Parametric Formal Model (The Engine)
You will create a robust PCSP# model for your chosen sport (e.g., tennis). Unlike previous
semesters, this model must be parametric.
● Instead of hardcoding values (e.g., serve T percentage/success rate), your model
must accept these as injectable parameters derived from real-world data.
● Goal: A template model that can represent any matchup just by changing the input
variables.
Part B: The Prediction Agent (Reachability Analysis)
Implement a pipeline where the user asks: "What is the expected winning percentage of
Sinner vs. Djokovic?"
● The LLM maps "Sinner" and "Djokovic" to their respective datasets.
● The system constructs the specific matchup.pcsp file.
● PAT checks the assertion: assert TieBreakGame reaches player1Win with prob;
● The system returns the precisely calculated probability to the user.
Part C: The Strategy Agent (Sensitivity Analysis)
Implement a pipeline for optimization questions: "What adjustment should Player A make to
beat Player B?"
● The Challenge: The LLM must define a search space. It should orchestrate a script
to "perturb" the model parameters (e.g., increase Serve_Wide% by 5%, decrease
Serve_T% by 5%).
● The Execution: Run the model checker on these variations to find the gradient (the
change that yields the biggest win-rate increase).
● The Output: Precise, quantified advice.
○ Bad Output: "Serve wider more often."
○ Good Output: "If you shift 20% of your T-serves to Wide-serves in the Deuce
court, your win probability increases by 4.2%."

5. Technical Implementation Tips
● Glue Code: You will likely need a Python wrapper. The LLM (e.g., OpenAI API or
Gemini API) will generate JSON instructions. Your Python script will parse this JSON,
edit the .pcsp text file, run the PAT console executable, and parse the output text.
● Prompt Engineering: You need to give the LLM "tools."
○ System Prompt Example: "You are an expert Sports Analyst. You have
access to a Formal Verification engine. Do not guess probabilities; use the
run_model_check tool to find them."
